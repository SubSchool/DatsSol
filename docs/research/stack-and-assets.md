# Stack And Assets

## Выбранный стек

### Backend

- FastAPI: быстрый API-слой, WebSocket для живого состояния и удобная интеграция с Pydantic.
- SQLAlchemy + Postgres: хранение логов, снапшотов тиков и ручных директив.
- Provider abstraction: позволяет переключать `datssol-mock` и `datssol-live` без переписывания runtime.
- DatsSol planning modules: отдельные слои `analyze`, `decide`, `execute`, чтобы оптимизировать эвристики по шагам.

### Frontend

- Vue 3 + Vite + TypeScript: быстрый цикл разработки и низкая цена изменений.
- PrimeVue: красивый набор компонентов для админки, таблиц, фильтров и панелей управления.
- PixiJS: рендеринг 2D-карты, сети ЦУ, boosted-клеток, прогнозов и маршрутов команд.
- `pan / zoom / drag` реализованы на чистом PixiJS, без дополнительной зависимости.

### Лёгкий 2D-движок на будущее

- Phaser 3 оставлен как опциональный кандидат, если позже понадобится полноценная `scene/game-loop` модель.
- `pixi-viewport` оставлен как опциональное усиление, если после первых тестов потребуется более сложная камера.

## Визуальное направление DatsSol

- Тема: пустыня, оазисы, инженерная сеть, ботаника поверх сухого рельефа.
- Цветовая логика:
  - песок и скалы для фона;
  - зелёный и бирюзовый для живых и терраформированных зон;
  - янтарный для boosted-клеток;
  - красный и карминовый для угроз и вражеских веток.
- На карте должны быть хорошо читаемы:
  - кардинальная связность с ЦУ;
  - boosted-клетки;
  - прогнозы катаклизмов;
  - маршруты ручных и автоматических команд.

## Ассеты и материалы

### Основные источники

- [Kenney Nature Kit](https://www.kenney.nl/assets/nature-kit): растительность, камни и декоративные элементы.
- [Kenney Roguelike Caves & Dungeons](https://www.kenney.nl/assets/roguelike-caves-dungeons): текстуры рельефа и скал для быстрых тайлов.
- [OpenGameArt desert tilesets](https://opengameart.org/): запасной источник пустынных и оазисных тайлов.
- Shape-based Pixi rendering остаётся дефолтом, чтобы не зависеть от ассетов в первые часы раунда.

### Практический подход

- На старте раунда использовать shape-based рендеринг и контрастную сетку для дебага.
- После первых тестовых раундов поверх shape-based слоя можно добавить декоративные спрайты оазисов, корней сети и бурь.
- Если конкурс визуализации будет важен, добавить:
  - анимированное прорастание терраформации;
  - тёплую песчаную дымку и пульсацию boosted-клеток;
  - трассы команд с разными цветами по типу действия;
  - миникарту и таймлайн последних угроз.
